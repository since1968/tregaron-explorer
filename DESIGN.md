# Tregaron Explorer — Design Brief

## Philosophy

This project is a **small, high-quality, interactive map** of Tregaron Conservancy.

It is not a general-purpose mapping tool.

Decisions should prioritize:
- clarity over completeness
- aesthetics over density
- usability over features

---

## Core Principle: Mobile First

This map is primarily used **on a phone, while walking in the park**.

All design decisions must optimize for:
- one-handed use
- quick glances (not long study)
- outdoor visibility (sunlight, glare)

Desktop is secondary.

---

## User Experience Goals

The map should feel:

- calm
- intuitive
- responsive
- lightweight

Users should never feel:
- overwhelmed
- lost
- confused by controls

---

## Visual Style

### General

- Clean, minimal interface
- Soft, natural color palette
- Avoid harsh contrast unless necessary

### Map Features

- Trails should be clearly visible but not overpowering
- Water features should feel calm and natural
- Landmarks should be subtle but discoverable

### UI Elements

- Rounded corners preferred
- Light shadows for separation
- Small, readable typography

---

## Interaction Design

- Interactions should feel **immediate and smooth**
- Animations should be **subtle, not flashy**
- Avoid unnecessary motion

### Popups

- Short, readable content
- No clutter
- Should not block key map areas

---

## Data Philosophy

- Prefer **accurate, minimal data** over noisy completeness
- Every feature on the map should have a purpose
- Avoid adding data that does not improve user understanding

---

## Performance

- Fast load times are critical
- Avoid heavy dependencies
- Keep everything client-side if possible

---

## Scope Discipline

Do NOT:
- Add features “just because we can”
- Generalize prematurely
- Turn this into a framework

Do:
- Build a polished, specific experience
- Expand only when it clearly improves the map

---

## Definition of Success

A user standing in Tregaron can:

- Open the map instantly
- Understand where they are
- Explore trails and features
- Enjoy using it

---

## Future Direction (Optional)

If expanded later, consider:
- ecological layers (flora/fauna)
- seasonal storytelling
- deeper terrain visualization

Only pursue these if they maintain simplicity.
